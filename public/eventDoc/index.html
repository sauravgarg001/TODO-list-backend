Events to listen

verifyUser event
This event ("verifyUser") has to be listened on the user's end to verify user authentication.
user will only be set as online user after verification of authentication token.
auth-error@<authToken> event
This event ("online-users") has to be listened on the user's end to identify authentication error or breach in security.
The output will error message.
disconnect event
This event ("disconnect") has to be listened, when the user is disconnected.
online-users event
This event ("online-users") has to be listened, to get list of online users when user logins.
The output will be array of emails of users.
online-user-add@<authToken> event
This event ("online-user-add@<authToken>") has to be listened, when the new online user is added.
The output will be email of user.
online-user-remove@<authToken> event
This event ("online-user-remove@<authToken>") has to be listened, when a online user disconnects.
The output will be email of user.
tasks@<authToken> event
This event ("tasks@<authToken>") has to be listened, when any user changes the list.
The output will updated tasks of list and listId.
add-contributer@<authToken> event
This event ("add-contributer@<authToken>") has to be listened, when any new contributer is added to list.
The output will contributer details and listId.
remove-contributer@<authToken> event
This event ("remove-contributer@<authToken>") has to be listened, when any contributer is removed from list.
The output will contributer email and listId.
change-contributer@<authToken> event
This event ("change-contributer@<authToken>") has to be listened, when any contributer access is changed in list.
The output will contributer email, canEdit, modifiedOn and listId.
add-friend@<authToken> event
This event ("add-friend@<authToken>") has to be listened, when any new friend has been added.
The output will friend details.
add-friend-request@<authToken> event
This event ("add-friend-request@<authToken>") has to be listened, when any new friend request has been arrived.
The output will friend request details.
remove-friend@<authToken> event
This event ("remove-friend@<authToken>") has to be listened, when any new friend has been removed.
The output will friend's email.
